# S-1 Grants Loop Standard

## BLUF
Grants are not a one-off writing task. Grants are a full S-1 loop:

```text
search → screen → score → select → write → QC → rewrite → QC → Commander approval → submit → verify → archive → adapt
```

S-1 may search, score, draft, QC, rewrite, and package autonomously. S-1 may submit only after Commander approval for the specific grant/submission and current-session credential authorization if credentials are required.

## Six-step learning loop

### 1. Observe
- Search live grant surfaces, prior inbox attachments, Drive, NOFOs, foundations, state/federal portals.
- Pull criteria, deadlines, eligibility, award size, reporting burden, application questions, attachments required.
- File source pages/PDFs and extracted text.

### 2. Learn
- Map opportunity against UAC mission:
  - heritage breed conservation,
  - Arapawa/CPL/rare livestock,
  - regenerative agriculture,
  - veteran/ag therapy,
  - infrastructure/equipment/capacity,
  - geography/eligibility.
- Identify missing proof, attachments, match requirements, letters, budgets, board/governance constraints.

### 3. Decide
- Score each opportunity:
  - mission fit,
  - eligibility fit,
  - evidence readiness,
  - effort/reporting burden,
  - deadline feasibility,
  - win likelihood.
- Classify:
  - `pursue_now`,
  - `research_more`,
  - `hold/archive`,
  - `not_fit`.

### 4. Act
For a selected grant:
1. draft narrative,
2. draft budget,
3. build attachment checklist,
4. QC against NOFO/rubric,
5. rewrite,
6. QC again,
7. produce Commander review packet.

### 5. Commander gate
Commander approves:
- final content,
- amount requested,
- commitments/reporting burden,
- submission authority,
- credential use if needed.

### 6. Submit / verify / adapt
After approval:
- submit through correct portal/email,
- capture confirmation number/email/screenshot/PDF,
- archive exact submitted package,
- create reporting/deadline tasks,
- track outcome,
- adapt scoring model based on wins/losses/feedback.

## QC standard
A grant is not shown to the Commander until it has passed:

- eligibility check,
- deadline check,
- mission fit check,
- budget arithmetic check,
- source-claim check,
- no UAC/TNR mixing check,
- no uncited animal/financial counts check,
- rubric alignment check,
- attachment completeness check.

## Real-world output standard
A grant loop output is one of:

- scored opportunity pipeline,
- Commander-ready application packet,
- submitted grant with verified confirmation,
- archive/reporting packet after submission.

A draft file alone is not completion.
